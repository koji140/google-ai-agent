# Google環境だけで作る業務エージェント実行基盤

## 目的

このプロジェクトで作りたいものは、開発者向けAI IDEやCLIエージェントではない。

目指すのは、Google Workspace上で発生する業務イベントを入口にして、AIが文書を読み、整理し、判断候補を作り、台帳に記録し、必要な通知や後続処理まで進める業務エージェント基盤である。

Claude CodeやCodexのような「考えて、作業して、記録する」体験を、コード開発ではなく業務運用に移す。ただし実行場所はローカル端末やIDEではなく、Google Drive / Sheets / Docs / Apps Script / Geminiを中心にする。

## 基本方針

- 業務ユーザーはGoogle Drive、Sheets、Docs、Gmail、Calendar、Chatから離れない
- AIへの依頼は毎回プロンプトを書く形にしない
- 入力元、処理内容、出力先、承認者、記録先を固定する
- 自動実行する前に、承認が必要な処理と不要な処理を分ける
- すべての判断、実行、失敗を台帳に残す
- 開発ツールは補助に留め、運用基盤の前提にしない

## 中核コンポーネント

| 役割 | Google製品 | 主な責務 |
|---|---|---|
| 入力 | Drive / Gmail / Forms / Calendar / Chat | 資料、議事録、依頼、予定、メッセージの発生源 |
| 状態管理 | Sheets | 台帳、ステータス、承認、実行ログ、エラー管理 |
| 文書保管 | Drive / Docs | 正本、議事録、成果物、レポートの保存先 |
| 実行 | Apps Script | トリガー、台帳更新、通知、API連携、軽量な自動処理 |
| 思考 | Gemini API / Gemini for Workspace | 要約、分類、抽出、判断案、次アクション生成 |
| 通知 | Gmail / Google Chat | 承認依頼、完了通知、期限リマインド、異常通知 |
| 監査 | Sheets / Docs | 判断ログ、実行履歴、週次サマリ、復旧記録 |

## 典型的な処理フロー

```text
1. Driveに資料が置かれる
2. Apps Scriptが時間トリガーで検知する
3. 文書台帳にファイルメタデータを登録する
4. Geminiが内容を要約し、種別や重要度を推定する
5. ToDo、決定事項、判断候補を台帳に分解する
6. 必要に応じて担当者へ承認依頼を送る
7. 承認された処理だけ実行する
8. 実行結果、判断理由、エラーをログに残す
```

## Antigravityなどの開発AIとの位置づけ

Antigravityのようなエージェント型IDEは、Apps ScriptやWebアプリを開発するときの支援ツールとしては有用である。

一方で、このプロジェクトの本番運用はIDEに依存しない。業務ユーザーが日常的に使う画面はGoogle Workspaceであり、エージェントの実行場所はApps ScriptとGoogle側のサービスである。

したがって、Antigravityは「作るための道具」にはなり得るが、「業務エージェントOSの本体」にはしない。

## Workspaceだけで実現する範囲

- 文書の自動台帳登録
- 議事録からのToDo抽出
- 決定事項と判断ログの記録
- 要約と分類の自動生成
- 期限リマインド
- 承認依頼と実行結果通知
- 週次サマリの生成
- 正本管理とステータス管理

## Workspaceだけでは弱い範囲

- 数十分以上かかる長時間処理
- 大量ファイルの高速な全文解析
- 複雑な外部Web操作
- 高度なキュー制御やリトライ制御
- GitHubのような厳密な差分レビュー
- ローカル端末上の自由なファイル操作

これらが必要になった場合は、Google Cloud Run、Cloud Scheduler、Pub/Sub、Workflows、Vertex AIなどに拡張する。ただし初期PoCではApps Script中心で始める。
