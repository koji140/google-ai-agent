# 次のアクション

## 現在のステータス

設計レポジトリの初期構築と、Phase 1向けの最小Apps Script実装が完了した段階。ここからGoogle Workspace上のPoC環境を作り、実際にDrive → Sheets同期を動かす。

---

## 最優先: PoCの環境準備

### アクション1: プロジェクト用Driveフォルダの作成

**やること**: Google Driveにプロジェクトルートフォルダを作成し、基本のフォルダ構造を構築する。

```
google-ai-agent-poc/
├── 01_打ち合わせ資料/
├── 02_議事録/
├── 03_HTML/
├── 04_成果物/
├── 05_判断ログ/
├── 06_テンプレート/
└── 99_台帳/
```

**所要時間**: 30分
**成果物**: フォルダ構造のスクリーンショット or URL

---

### アクション2: 文書台帳Sheetの作成

**やること**: Google Sheetsで文書台帳を作成する。

列構成:
| 列 | 内容 |
|----|------|
| A | ファイルID（Apps Scriptで自動取得） |
| B | ファイル名 |
| C | 種別（打ち合わせ資料/議事録/HTML/成果物/判断ログ） |
| D | 作成日 |
| E | 更新日 |
| F | フォルダパス |
| G | URL |
| H | 要約（Geminiで自動生成。Phase 1では空欄） |
| I | ステータス（ドラフト/確定/廃止） |
| J | 登録方法（自動/手動） |
| K | 備考 |

**所要時間**: 1時間（列設計 + フォーマット設定 + サンプルデータ入力）
**成果物**: 文書台帳SheetのURL

---

### アクション3: 最小限のApps ScriptをApps Scriptプロジェクトへ反映

**やること**: リポジトリに追加済みのDrive監視 → Sheets登録スクリプトを、Apps Scriptプロジェクトへ反映する。

最小限の仕様:
1. 指定フォルダのファイル一覧を取得
2. 台帳に存在しないファイルを検出
3. 新規ファイルを台帳に行追加
4. 1時間ごとの時間トリガーを設定

**所要時間**: 30分〜1時間（clasp設定 + 初回承認 + テスト）
**成果物**: Google Workspace上で動作するApps Script

補足:
- ローカルの実装は `apps-script/src/Code.js` にある
- 手順は `docs/development-setup.md` を参照する
- 次の改善として、実行ログSheetとエラーログSheetを追加する

- ...

- [ ] [担当者] [タスク内容] [期限: yyyy-mm-dd]

## 判断事項

- 判断理由: ...

**所要時間**: 1時間
**成果物**: 議事録テンプレートDocのURL

## その後: Gemini API連携

### アクション6: Gemini APIの接続確認

**やること**: Apps ScriptからGemini APIを呼び出せることを確認する。

手順:
1. Gemini APIキーを取得
2. Apps Scriptのプロパティサービスにキーを格納
3. `UrlFetchApp.fetch()` でテスト呼び出し
4. レスポンスの確認

**所要時間**: 1〜2時間
**成果物**: API接続が確認できたスクリプト

---

### アクション7: 議事録抽出プロンプトの設計

**やること**: 議事録からToDo・決定事項・要約を抽出するプロンプトを設計する。

検討事項:

**所要時間**: 2〜3時間（プロンプト設計 + テスト）
**成果物**: プロンプトテンプレート + テスト結果

## PoCの成功基準

Phase 1のPoCとして、以下が実現できれば成功とする:

1. Driveにファイルを置いたら、1時間以内に台帳に自動登録される
2. 台帳の内容が正確である（ファイル名、日付、URL等）
3. 重複登録がない
4. 手動で台帳を更新する必要がない
5. 実行結果とエラーを後から確認できる

この段階では、Gemini連携は不要。Drive + Sheets + Apps Scriptの3つで動く最小構成を確認する。
